# Proxying Homebox
Homebox is served by two upstreams that must share a single origin:

- The **API** (default port `7745`) handles `/api/*` (including the `/api/v1/ws/events` WebSocket) and `/swagger/*`.
- The **frontend** (default port `3000`) server-side renders every other path.

Your reverse proxy must route by path prefix to the correct upstream.

> [!WARNING]
> When running Homebox behind a reverse proxy, you must set `HBOX_OPTIONS_TRUST_PROXY` to `true` in your API environment variables/configuration.
> This ensures that the application correctly processes forwarded headers (X-Forwarded-For, X-Forwarded-Proto, etc.) from your proxy.
> Without this, login cookies, labels and some other functionality may not work properly.

The examples below use `localhost:7745` for the API and `localhost:3000` for the frontend; replace them with your
container names or hosts (e.g. `homebox-api:7745` and `homebox-frontend:3000` in Docker Compose).

## Nginx

Nginx requires proper configuration to handle WebSocket upgrades and forward appropriate headers to Homebox.

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location ~ ^/(api|swagger)/ {
        proxy_pass http://localhost:7745;
        proxy_http_version 1.1;

        # WebSocket support (/api/v1/ws/events)
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # Essential proxy headers
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Timeouts for long-lived WebSocket connections
        proxy_read_timeout 86400;
    }

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### HTTPS Example
```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location ~ ^/(api|swagger)/ {
        proxy_pass http://localhost:7745;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Traefik

Traefik handles WebSocket connections automatically. Here's a configuration using labels (Docker Compose) and YAML.

### Docker Compose Labels
```yaml
services:
  homebox-api:
    image: ghcr.io/sysadminsmedia/homebox:latest
    environment:
      - HBOX_OPTIONS_TRUST_PROXY=true
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.homebox-api.rule=Host(`your-domain.com`) && (PathPrefix(`/api`) || PathPrefix(`/swagger`))"
      - "traefik.http.routers.homebox-api.entrypoints=web"
      - "traefik.http.services.homebox-api.loadbalancer.server.port=7745"

  homebox-frontend:
    image: ghcr.io/sysadminsmedia/homebox-frontend:latest
    environment:
      - NUXT_API_HOST=http://homebox-api:7745
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.homebox-frontend.rule=Host(`your-domain.com`)"
      - "traefik.http.routers.homebox-frontend.entrypoints=web"
      - "traefik.http.services.homebox-frontend.loadbalancer.server.port=3000"
```

Traefik picks the API router for `/api` and `/swagger` automatically because its rule is more specific.

### YAML Configuration
```yaml
http:
  routers:
    homebox-api:
      rule: "Host(`your-domain.com`) && (PathPrefix(`/api`) || PathPrefix(`/swagger`))"
      service: homebox-api
      entryPoints:
        - web

    homebox-frontend:
      rule: "Host(`your-domain.com`)"
      service: homebox-frontend
      entryPoints:
        - web

  services:
    homebox-api:
      loadBalancer:
        servers:
          - url: "http://localhost:7745"

    homebox-frontend:
      loadBalancer:
        servers:
          - url: "http://localhost:3000"
```

### HTTPS with Let's Encrypt
```yaml
http:
  routers:
    homebox-api:
      rule: "Host(`your-domain.com`) && (PathPrefix(`/api`) || PathPrefix(`/swagger`))"
      service: homebox-api
      entryPoints:
        - websecure
      tls:
        certResolver: letsencrypt

    homebox-frontend:
      rule: "Host(`your-domain.com`)"
      service: homebox-frontend
      entryPoints:
        - websecure
      tls:
        certResolver: letsencrypt

  services:
    homebox-api:
      loadBalancer:
        servers:
          - url: "http://localhost:7745"

    homebox-frontend:
      loadBalancer:
        servers:
          - url: "http://localhost:3000"

certificatesResolvers:
  letsencrypt:
    acme:
      email: your-email@example.com
      storage: /data/acme.json
      httpChallenge:
        entryPoint: web
```

## Caddy

Caddy automatically handles WebSocket upgrades and HTTPS with Let's Encrypt certificates.

```caddy
your-domain.com {
    handle /api/* {
        reverse_proxy localhost:7745
    }
    handle /swagger/* {
        reverse_proxy localhost:7745
    }
    handle {
        reverse_proxy localhost:3000
    }
}
```

### Caddyfile with Custom Configuration
```caddy
your-domain.com {
    encode gzip

    handle /api/* {
        reverse_proxy localhost:7745 {
            header_up X-Forwarded-Proto {scheme}
            header_up X-Forwarded-For {remote_host}
            header_up X-Real-IP {remote_host}
        }
    }
    handle /swagger/* {
        reverse_proxy localhost:7745
    }
    handle {
        reverse_proxy localhost:3000
    }
}
```

## WebSocket Support

All configurations above support WebSocket connections to the `/api/v1/ws/events` endpoint (routed to the API upstream). Key requirements for WebSocket support:

- **Nginx**: `proxy_set_header Upgrade` and `proxy_set_header Connection "upgrade"` on the API location
- **Apache2**: WebSocket rewrite rules with `RewriteCond %{HTTP:Upgrade}`
- **Traefik**: Automatic support (no special configuration needed)
- **Caddy**: Automatic support (no special configuration needed)

## Troubleshooting

### WebSocket Connection Fails
- Verify the proxy properly forwards the `Upgrade` header on the `/api` route
- Check that `Connection: Upgrade` header is being forwarded
- Ensure timeouts aren't closing long-lived connections

### Incorrect User IP or HTTPS Detection
- Confirm `HBOX_OPTIONS_TRUST_PROXY=true` is set on the API
- Verify the proxy sends `X-Forwarded-For` and `X-Forwarded-Proto` headers
- Check firewall rules allow connections between proxy and Homebox

### 502 Bad Gateway
- Ensure the API is running on port 7745 and the frontend on port 3000 (or your configured ports)
- Verify the proxy can reach both containers/services
- Check proxy logs for connection errors

### Pages Load but API Calls Fail
- Confirm `/api` and `/swagger` are routed to the API upstream, not the frontend
- Confirm the frontend's `NUXT_API_HOST` points at the API (used for server-side rendering)
